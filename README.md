## ▶ HOW TO PLAY

```bash
npm install
npm start
```

Open **http://localhost:3000** — that's the whole game.

### ⚔ CONTROLS

| ACTION | KEYS |
|:-------|:-----|
| **Move** | `W` `A` `S` `D` / arrows |
| **Attack** | `J` / left-click · *(Ranger: hold to charge)* |
| **Class Skill** | `K` / right-click — *Knight: hold-block → tap-parry · Mage: Nova · Ranger: volley* |
| **Dodge / Blink** | `Space` — *brief invincibility frames* |
| **Revive / Interact** | `E` — *hold near a downed ally* |
| **Advance Dialogue** | click / `Enter` |
| **Menu** | `Esc` |

> Mage and Ranger aim with the **mouse**.

## 🎮 PLAY WITH FRIENDS

One player opens a **Campaign** (1–3 players) or a **Duel** (1v1), then shares the **4-letter room code** from the lobby. Friends type it into *Join a Friend*. The host picks the mission and sounds the horn.

- **🌐 Over the internet** — deploy to Render (build `npm install`, start `node server.js`). WebSockets and `wss://` are handled automatically.
- **📡 Same Wi-Fi** — find your LAN IP (`ipconfig getifaddr en0`) and share `http://<your-ip>:3000`.
- **🕳️ Tunnel** — `ngrok http 3000`.

---

## 💰 PROGRESSION & ECONOMY

Coins from kills, mission rewards, and duels are saved **per browser** and spent in the **Armory** on weapons, armor, stat training, and cosmetics. Story missions unlock in order. Every fight makes your hero visibly stronger — and every hero is fully **customizable** (colors, headwear, capes, ember trails) in both modes.

```
  ◈ ARMORY ────────────────────────────────────────────────────
    ⚔ Weapons   ->  Ember Brand · Dawnbreaker · Heart of the Choir
    🛡 Armor     ->  Vigil Plate · Sanctum Aegis
    ↑ Training   ->  Vitality · Might · Swiftness · Arcana
    ✦ Cosmetics  ->  Vigil Cape · Ember Trail · Headwear
  ──────────────────────────────────────────────────────────────
```

## 🏗 UNDER THE HOOD

```
  server.js      ->  static host + WebSocket relay. Rooms, heartbeats,
                     host migration. No game logic on the server.

  public/game.js ->  the whole game. Each client simulates its own hero
                     (so your dodge/parry is judged locally — fair under
                     lag). The room host simulates enemies & missions and
                     broadcasts snapshots ~10 Hz. If the host drops, another
                     player adopts the simulation mid-mission.
```

Netcode is trust-based and favors **stability over frame-perfect sync** by design — built to be lag-tolerant for playing with friends across any connection.
